# DADS Design System JA

デジタル庁デザインシステム（DADS）の公式情報を必要な範囲だけ参照し、日本語の公共・業務サービスUIへ適用する。これはデジタル庁が提供する公式Skillではない。公式原文とローカル独自拡張を混同しない。

## 適用対象を判定する

- DADSの導入・準拠状況確認、公共・行政サービス、信頼性を重視する業務システムに適用する。
- 日本語フォーム、入力エラー、テーブル、検索・絞り込み、ナビゲーション、アクセシビリティ確認に適用する。
- HTML版・React版コンポーネント、Tailwindテーマの導入に適用する。
- DADSの思想を応用した硬い業務資料に適用する。
- 音楽、アート、MV、ジャケット、物語、娯楽中心のサイト、感情表現や世界観を最優先する制作物には適用しない。
- DADSと関係しない一般的なバックエンド作業には適用しない。

## 作業モードを選ぶ

依頼を次のいずれかに分類し、回答で選択したモードを明示する。

1. **公式DADSの実装**: 公式ガイドラインと公式コードを優先し、公式との差異を明示する。
2. **DADSを参考にしたカスタマイズ**: サービス固有のブランドや要件に合わせ、DADS公式制作物であるかのように表現しない。
3. **既存画面の監査**: アクセシビリティ、情報構造、操作性、コンポーネント利用を確認し、「問題・公式根拠・改善案・未検証」を分ける。
4. **業務資料への応用**: 文字、色、余白、情報階層、状態表現を応用し、DADS公式文書テンプレートとは表現しない。

判断が曖昧な場合は [decision-guide.md](references/decision-guide.md) を読む。業務資料では [document-adaptation.md](references/document-adaptation.md) を追加で読む。

## 基本ワークフロー

1. 対象の目的、利用者、利用環境を確認する。
2. 実装対象では、リポジトリの `AGENTS.md`、技術構成、既存デザインシステム、依存関係とバージョンを先に確認する。
3. [official-index.md](references/official-index.md) から該当する公式Markdownだけを読む。全公式Markdownを毎回読み込まない。
4. 使用するコンポーネントとデザイントークンを選ぶ。
5. キーボード操作、フォーカス、コントラスト、エラー通知、拡大・リフロー等のアクセシビリティ要件を先に確認する。
6. DADSをそのまま使う範囲と、既存ブランドへ適合させる範囲を分ける。
7. 最小限の変更で設計または実装する。既存バージョンや依存関係を無断で変更しない。
8. 対象に応じて自動テストと手動アクセシビリティ確認を行う。
9. 公式DADS由来の部分、独自変更、出典・ライセンス、実施できなかった検証を報告する。

実装時は [implementation-guide.md](references/implementation-guide.md)、コードや素材を利用するときは [licensing.md](references/licensing.md) を読む。

## 公式情報を参照する

- ローカル公式原文は `references/official/` に格納する。原文を編集せず、独自ルールを混ぜない。
- 色・文字・余白は `foundations/`、フォーム等は対象の `components/` とアクセシビリティ文書を読む。
- Reactでは公式Reactリポジトリと対象コンポーネント、Tailwindでは公式テーマと対象プロジェクトのバージョンを確認する。
- HTML・React・Tailwindの最新実装事実、現行バージョン、ライセンスが重要な場合は公式サイトまたは公式リポジトリを再確認する。
- ローカル配布物は検索用スナップショットとして扱い、完全な一次情報が必要な場合は各Markdownの `source_url` を確認する。

## 更新と検証

公式情報の更新を明示的に依頼された場合だけ、次を実行する。

```text
python scripts/update_official_sources.py
python scripts/validate_skill.py
```

通常のSkill利用時に自動更新しない。取得元、版、SHA-256、確認日は [source-manifest.md](references/source-manifest.md) で確認する。

## 禁止事項

- デジタル庁公式Skill、公式認定、公式テンプレートであると誤認させない。
- デジタル庁のロゴや公式ブランドをSkillのアイコンに使わない。
- 公式Markdown内へ独自要約や応用ルールを書き込まない。
- 公式コード、Figma、イラスト・アイコンの利用条件を推測しない。
- 対象リポジトリの既存技術、`AGENTS.md`、アクセシビリティ要件を確認せず導入しない。
