# DADS適用の意思決定ガイド

この文書は公式DADSの原文ではなく、公式情報を作業へ適用するためのローカル独自ガイドです。

## 適用モードを決める

| 状況 | 推奨モード | 判断 |
|---|---|---|
| 行政・公共サービスでDADSの仕様を採用する | 公式DADSの実装 | 公式ガイドラインと公式コードを優先し、差異を記録する |
| 既存ブランドや固有要件がある | DADS参考のカスタマイズ | DADSを骨格に使い、サービス固有スタイルガイドへ変換する |
| 既存画面の品質確認 | 既存画面の監査 | 問題、公式根拠、改善案、未検証を分ける |
| レポートや提案書など非Web成果物 | 業務資料への応用 | DADSの情報階層・可読性・状態表現を応用し、公式資料とは呼ばない |

## そのまま使用するか変換するか

DADSをそのまま使用するのは、対象が公共性の高い日本語サービスで、既存ブランドと技術構成が適合し、DADSの仕様を維持できる場合とする。

次の場合はサービス固有のスタイルガイドへ変換する。

- 既存ブランドの色、書体、語調、コンポーネント体系との整合が必要
- 対象利用者、業務フロー、端末、運用要件がDADS作例と異なる
- 一部コンポーネントだけを段階導入する
- DADSの更新を追従する責任者と運用手順を定める必要がある

変換後は「DADS準拠」と断定せず、「DADSを参考に設計」「DADSのガイドラインを一部採用」など、実態に合う表現を使う。

## 公式コードを使うか参考実装にするか

公式コードを利用する前に、対象リポジトリのフレームワーク、バージョン、CSS戦略、ライセンス、テスト、アクセシビリティ要件を確認する。

次の場合はコピーではなく参考実装を選ぶ。

- 公式サンプルのReact・Tailwind等のバージョンと対象が異なる
- 既存コンポーネントAPIやデザイントークンへ統合する必要がある
- 公式サンプルの依存関係を追加できない
- 必要な振る舞いだけを既存設計に合わせて再現する方が保守しやすい

## HTML版とReact版を選ぶ

- プレーンHTML／CSS／JavaScript、静的サイト、フレームワーク非依存ならHTML版を参照する。
- React／TypeScriptの既存プロジェクトならReact版を参照する。
- React版を選ぶだけで互換性が保証されるとは考えない。公式サンプルの現行技術スタックと対象の差を確認する。
- Storybookは振る舞いと作例の確認に使い、対象リポジトリの品質基準とテストを置き換えない。

## Tailwind CSS v3とv4を扱う

公式Tailwindテーマはv3とv4の導入方法を案内しているが、設定形式は異なる。対象プロジェクトのメジャーバージョンを確認し、該当する公式手順だけを使う。アップグレードを導入作業へ混ぜず、必要なら別変更として提案する。

## Reactのバージョン差を扱う

2026-07-16確認時点で、公式ReactサンプルはReact 18を基礎とし、React 19では軽微な型エラーの可能性を案内している。利用時点の公式READMEと対象のバージョンを再確認し、型・イベント・ref・テストの差を実測する。

## 誤認を防ぐ

- 「デジタル庁公式Skill」「デジタル庁認定」「公式テンプレート」と表現しない。
- 独自変更後の成果物を「DADS公式コンポーネント」と呼ばない。
- DADSの原文・公式コード・独自実装を成果報告で区別する。
- デジタル庁のロゴや公式ブランドをSkillや成果物の出所表示に流用しない。

## アイコンを加工する

イラスト・アイコン素材を編集・加工して公表、利用、再配布する場合は、公式利用規約を確認し、出典と加工内容を明示する。加工物を国または府省等が作成したものと誤認させない。個別素材に特記がある場合は特記を優先する。
